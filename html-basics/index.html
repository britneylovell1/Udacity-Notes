HTML and CSS Basics

I. Components of the Web

i. The Web

There are four major pieces to the World Wide Web. The internet is a network of computers that communicate to each other. When you want to access a website, your computer communicates to other computers by sending a HTTP request to their server. The correct HTML document is sent back to your computer using the browser to display it.

ii. HTML

HTML stands for Hypertext Markup Language. HTML documents are what make up most of the internet. These documents are comprised of text and tags that tells the browser how to project the text to the user. It is often in the form <tag attribute="value">. HTML files determines the tree-like structure of the webpage.

iii. CSS Files

CSS stands for Cascading Style Sheets. As the name suggests, CSS Files determines the style of the webpage. A CSS file is a separate document (labeled "insertnamehere.css") and is found in the same folder as the HTML file that we want to style. The CSS document is made up of selectors that refer to the class name of the element that we want to style. The selector tells the browser which element the rule (or declaration) inside the curly braces applies to. 

            selector {
              property: value;
            }

For example, 

            h1 {
              background-color: black;
              color: white;
            }

tells the browser that all h1 elements have a black background and white text.

iv. Java Script Files

Java Script Files determine the interactive components in a webpage.

II. The Process of Web Development

i. Helpful Tools

These tools allow you to see how a certain webpage was constructed. When using Google Chrome, click on tools > developer tools. This allows you to look at code for the entire html page. You can also click on an element on the webpage and press control > Inspect Element to access code for that specific element.

ii. Outlining Your Web Page

Start by placing "boxes" on your website grid. Decide where you want your main ideas to be placed on your webpage. Create div tags for each box (<div></div>). Give each box a class name (<div class=“title”>).

iii. Adding Style

There are three options to adding style:

  1. You can add style with the style tag in HTML in the head. For example, 
                    <head>
                  <style>
                    div{
                      color: blue
                    }
                  </style>
                </head>
    this tells the browser to make all text within div tags blue.
  2. You can use the style attribute in HTML. This is written inline within the body of the HTML doc. For example, 
                    <body>
                  <div style="color : blue">This text would be blue.</div>
                </body>
    this tells the browser to make the text within those specific div tags the color blue.
  3. We can also add style by refering our HTML Document to a CSS file in the same folder as our HTML doc. This is typically the ideal way of creating style. Let this be an HTML document called main.html. We will call the CSS document that we link to, main.css. 
                    <!DOCTYPE html>
                   This tells the browser the type of document 
                <html>
                   Everything between the html tags is in html 
                  <head>
                     This is where you add the style file. It often looks like this: 
                    <link rel="stylesheet" href="main.css" />
                  </head>
                  <body>
                     This is where you put the html structure 
                  </body>
                </html>
    The CSS document will look like: 
                    div {
                  background-color: red; 
                }

iv. The Box Model

Recall that HTML documents are made up of boxes. Each box has 4 components: content, padding, border, and margin. Sizing becomes a bit of a challenge considering all four components. We can approach this problem from two ways:

  1. Set sizes in terms of percentages rather than pixels.
  2. Set the box-sizing attribute to border-box for every element.
    *{box-sizing: border-box;}

To reposition boxes, we need to resize them. Since divs are a block element, they will take up the width of the entire page. Using the rule "display: flex;" will allow the div boxes to appear next to each other.

Working in Python

I. Variables and Strings

In computer science, variables represent values. A value, or expression, is assigned to a variable name with an assignment statement. 

            <variable name> = <expression>

It is important to note that "=" does not mean "equals to", rather it is the operator that assigns the right side to the left side.

A string is a sequence of characters surrounded by double or single quotes. Strings can be added, or concatenated, and multiplied. If 

            s = "Hello! "
            t = "How are you?"

Then, 

            s + t = "Hello! How are you?"
            s * 3 = "Hello! Hello! Hello! "

We can also index strings, using the form <string>[<starting expression> : <ending expression>]. For example, if 

            s = "Computer Cat"

Then, 

            <s>[0] = "C",
            <s>[1 + 1 : 10/2] = "mpu"

II. Functions

We define a function as such: 

            def <function_name>(<parameters>):
              '''
              function description
              '''
              <block>
              return <expression>

Parameters are passed through the function. The function does some calculations on the parameters and spits back the output. Defining a function is only the act of making a function. Calling a function is the actual use of the function. We enter in the specific parameters necessary for a program in place of the generalized parameters we created when defining the function. The return statement is what ends the function and spits back the output. If no return statement is present, the function passes back 'None'. For example, 

            def <create_an_animal(animal_1,animal_2):
              '''
              Concatenate two animals.
              Return new animal.
              '''
              new_animal = animal_1 + animal_2
              return new_animal
              
            print create_an_animal('cat','llama')

The result of running this program would be: 

            catllama

III. Control Flow

i. While Loops

We define a while loop as such: 

              while <test_expression>:
                <block>

If the test expression is True, then the block of code is executed over and over, until the test expression is False. Sometimes a break statement is needed to exit the while loop. 

              while <test_expression>:
                <block>
                if <test_expression>:
                  break
                <block>
              <after_while_loop>

ii. For Loops

We define a for loop as such: 

              for <name> in <list>:
                <block>

A for loop goes through each element in the list in order and stops after executing the block of code for the last element in the list.

iii. If Statements

We define an if statement as such: 

              if <test_expression>:
                <block>

If the test expression is true, then the block of code is executed. If not, then the block of code is skipped over. To test multiple cases, include elif statements and/or else statements. 

              if <test_expression_1>:
                <block>
              elif <test_expression_2>:
                <block>
              else: 
                <block>

The test expression in the if statement always returns a Boolean value - True or False. Sometimes operands - < , >, ==, != - are used to decide which Boolean value needs to be returned.

An or statement can also be used in the test expression. We define an or statement as such: 

              <expression> or <expression>

If the first expression is True, then the second expression is not evaluated. However, if the first expression if False, the second expression is evaluated. It only takes one True expression in an or statement to produce a True result. If both expressions are False, then the or statement returns False.

IV. Structured Data

i. Lists

We define a list as such: 

            <list_name> = [<expression>, <expression>, ...]

Similar to strings, lists can be accessed using the index. The main difference, however, between strings and lists is that a list is mutable. Strings, themselves, do not change, but lists can! For example, let's say 

            some_string = 'hello world'

Let's say that we decide to change some_string to a different string. For instance, 

            some_string = "good-bye y'all"

The actual string, 'hello world', has not changed. It is only the variable that it was assigned to has changed. 'hello world' is still in the computer memory. When we reference some_string, it pulls up "good-bye, y'all" instead of 'hello world'. Let's say, 

            some_list = ['a','b', 1 ,2]

Let's say we want to add 'yes' and 'no'. For instance, 

            some_list = some_list + ['yes', 'no']

Now some_list has changed. There is no such list called some_list with ['a','b',1,2] as its elements anymore in the computer's memory. It has changed to contain ['a','b',1 ,2 , 'yes', 'no']. This becomes important when we try to assign another variable to the new list we've created. Let's say that another_list is also assigned to ['a','b',1 ,2 , 'yes', 'no']. Then, some_list and another_list are equal to each other. If we change one of the lists, then the other changes. For example, if 

            some_list[0] = 'uh-oh'

Then some_list = ['uh-oh','b',1 ,2 , 'yes', 'no'] and another_list = ['uh-oh','b',1 ,2 , 'yes', 'no']. Because the list, itself, has changed, then both the variables assigned to the list are still equal to each other.

We can add or append elements to a list. Adding elements with '+' simply adds each single element to the end of a list. Appending elements perserves the structure of the elements being appended. For example, 

            print some_list + ['maybe', 'okay']
            print some_list.append([3, 4])

would yield this result: 

            ['uh-oh','b',1 ,2 , 'yes', 'no', 'maybe', 'okay']
            ['uh-oh','b',1 ,2 , 'yes', 'no', 'maybe', 'okay', [3,4]]

V. Some Tips for Debugging

  1. Check the error message for details on the error and line number.
  2. When working with example code, make sure the code works. Then make minor adjustments to fit your need.
  3. Include print statements at important steps to check how the code is executed.
  4. Don't be quick to throw out code. Comment out the code that may be causing the error. It may be useful later.
  5. Create comments that are up to date and concise, but don't comment obvious code. Be sure to include comments before functions.

VI. Problem Solving Strategies

  1. Understand the inputs of the problem.
  2. Understand the outputs of the problem.
  3. Understand the procedure. Work out some examples!
  4. Think about how to solve the problem in a humanly systematic way. Write it as pseudocode. The simpler, the better!
  5. Don't optimize prematurely.
  6. Test code often and in small chunks.

VII. Libraries, Files, and Classes

The Python Standard Libary contains different files filled with pre-made functions. We can think of the Python Standard Library as a filing cabinet. Inside the filing cabinet are a bunch of folders organized by content. Inside each folder, or file, are related functions. To use these functions in a python program, we need to access the file within the Python Standard Library by importing it, as such: 

            import webbrowser
            
            webbrowser.open("webpage_address")

This program opens up a webpage by calling the function, open(), which is found within the file, webbrowser, inside the Python Standard Library. Not having to see the actual code for the function open() is called abstraction and allows us to focus on writing the code we want without having to worry about every single detail. Go and explore more files and functions in the Python Standard Library! The Python Standard Library also contains built-in functions that don't require the importation of a file.

VIII. Classes

i. What is a class?

A class is a set of instructions, like a blueprint, found within a file. We can create distinct objects, or instances, of a class using the instructions within the class. As a blueprint for a house can present a house with yellow walls and another house with blue walls, so a class can present similiar but distinct objects.

Although the Python Standard Library has much to offer, there are some packages that are available outside the PSL that we can download. Twilio, for instance, is a package that sends text messages. 

            from twilio import rest
            
            client = rest.TwilioRestClient(account_sid, auth_token)

This is only a snippit of the actual code that allows us to send text messages. In this code, client is an object of the class, TwilioRestClient. This class is found inside the file, rest, which is inside the file, twilio. So this code says, "from the file called twilio, import the file called rest so that we can use the class, TwilioRestClient."

ii. Creating a Class

Let's create a python document called, classDocument.py and within it, let's define a class as such: 

            class ClassName():
              <statement-1>
              <statement-2>
              ...
              <statement-N>

A class contains statements, which can be both class variables and methods. Class variables are just variables defined with the class and methods are just functions defined within the class. Commonly found within a class is the class variable, __doc__, which provides information on how the class works. This class variable is usually declared as such: 

            '''This is what the class does.'''

When we call it in this way, 

            classDocument.ClassName.__doc__

the output will be This is what the class does.

Also commonly found within a class is the class constructor, also known as the __init__ function, and it is defined as such: 

            def __init__(self, argument1, argument2, ..., argumentN):
              self.name1 = argument1
              self.name2 = argument2
              ...
              self.nameN = argumentN

The instances created in defining the class, ClassName, are defined in a separate python document, but saved in the same directory as classDocument.py. We create an instance as such: 

            import classDocument
            
            instance_1 = classDocument.ClassName(argument1, argument2, ..., argumentN)

All the variables associated with an instance is called instance variables.
